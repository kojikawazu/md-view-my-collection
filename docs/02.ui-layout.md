# 最新UIレイアウト要求（base準拠）

※ `docs/01.pre-requirements.md` は読み取り専用。以降の要求反映は本書に追記/更新する。

## 概要
- `base/` の画面実装が最新のレイアウト定義。
- 対象は「Markdownレポート閲覧・管理UI」。
- 画面は一覧 / 詳細 / 新規作成 / 編集 / ログインで構成される。

## グローバルレイアウト
- ヘッダー＋サイドバー＋メイン＋フッターの2カラム構成。
- ヘッダーは固定表示（sticky）で、未ログイン時はログインボタン、ログイン時は「New Post」とユーザー情報＋Logoutを表示。
- サイドバーはPC表示のみ（`md`以上）でカテゴリとトレンドタグ、編集者メモを表示。
- ログイン画面のみ、ヘッダー/サイドバー/フッターを非表示で全画面センタリング。

## 画面: 一覧（/）
- 画面上部にタイトル「Latest Reports」と説明文。
- レポートは2カラムカード（`lg`以上）で表示。
  - カテゴリバッジ、日付、タイトル、要約、著者名を含む。
  - 「Read Report →」リンクで詳細へ遷移。
- レポートが0件の場合は「No reports found.」を中央表示。

## 画面: 詳細（/report/:id）
- パンくず的な「All Reports」リンクとカテゴリバッジを表示。
- タイトル、著者名、日付を表示。
- ログイン時のみ「編集」「削除」アクションを表示。
- 本文はMarkdownライクな簡易レンダリング（見出し/リスト/改行）。
- タグはフッター領域でチップ表示。
- 削除は確認モーダルを挟む。

## 画面: 新規作成・編集（/report/new, /report/:id/edit）
- フォーム項目
  - タイトル、カテゴリー、タグ（カンマ区切り）、要約、本文（Markdown）
- 送信前に確認モーダルを表示。
- 未ログイン時はログイン画面へリダイレクト。

## 画面: ログイン（/login）
- ユーザー名とパスワード入力、Authenticateボタン。
- 入力が揃えばログイン成功（現状は簡易認証）。
- 文言は英語、コンポーネントはカード中央配置。

## UIトーン/スタイル
- 深いブラウン系のハイコントラストデザイン。
- 見出しは太字・大きめ、本文は落ち着いた書体。
- ボタンは太字・大文字・間隔広めのラベル。

## データ/状態（現行の挙動）
- レポートとログイン情報は `localStorage` に保存。
- 初期データは `base/constants.tsx` の `INITIAL_REPORTS` を使用。

## ルーティング
- `/` 一覧
- `/report/:id` 詳細
- `/report/new` 新規作成
- `/report/:id/edit` 編集
- `/login` ログイン

## 差分（01.pre-requirements.md → 最新UI）
- アーキテクチャ想定（Next.js/TS/TailwindCSS/Supabase）に対し、現状はVite + Reactのフロント単体で画面レイアウトを先行。
- レポート閲覧中心の要件は維持しつつ、ログイン/投稿/編集/削除までのUIをフロントで具現化。
- Markdown表示はサニタイズ済みの外部ライブラリではなく、簡易パーサでの表現に留める。
- 公開範囲/権限（public/auth/admin）は将来想定だが、現状はログイン有無のみで制御。
- 画面構成は一覧/詳細/投稿/編集/ログインの5画面に確定。
